pub struct NodeManager {
    nodes: HashMap<String, Node>,
    links: HashMap<String, Recipient<ConsumerMessage>>,
    consumers: HashMap<String, Recipient<ConsumerMessage>>,
    producers: HashMap<String, Recipient<GetProducerMessage>>,
    no_more_modes_sender: Option<Sender<()>>,
    listeners: HashMap<String, WeakRecipient<NodeStatusMessage>>,
}

NodeManager acts as a tracker of all nodes, and dispatches messages accordingly

Nodes can be producers, consumers or both. NodeManager knows how to make logical links from one to another, actual connection to producers is delegated to the consumers however, as they might want to only perform the connection once their state has progressed.

Links have an identifier (provided by the client), that identifier can be used by the client to perform disconnection, and are used by consumers that expose multiple consumer slots to identify the correct slot to disconnect (eg mixers)

no_more_modes_sender: Option<Sender<()>>

Used when the manager is “stopping”, to ensure all nodes have fully stopped before exiting. Useful for propagating EOS.

Listeners for unit tests and potential user interfaces. Listener id -> recipient
